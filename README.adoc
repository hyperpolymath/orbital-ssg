== Overview

*orbital-ssg* is a collection of 28 MCP-compatible adapters for static site generators across 17 programming languages. Part of the https://github.com/hyperpolymath[hyperpolymath] ecosystem.

[.lead]
Enable AI agents to build, serve, and manage static sites across Rust, Haskell, Julia, Elixir, and 13 more language ecosystems.

=== Features

* *28 SSG Adapters* - Zola, Hakyll, Franklin, Serum, and more
* *17 Languages* - Rust, Haskell, Julia, Clojure, Elixir, Scala, etc.
* *MCP Protocol* - Model Context Protocol compatible tools
* *Secure by Design* - Input sanitization prevents code injection
* *Deno Runtime* - Modern, secure JavaScript runtime
* *RSR Compliant* - Follows Rhodium Standard Repository guidelines

=== Quick Start

[source,bash]
----
# Clone the repository
git clone https://github.com/hyperpolymath/orbital-ssg.git
cd orbital-ssg

# Install dependencies
asdf install

# Run tests
just test

# List adapters
just adapters
----

== Usage

=== As MCP Tools

Each adapter exports MCP-compatible tools:

[source,javascript]
----
import * as zola from "./adapters/zola.js";

// Check connection
if (await zola.connect()) {
  // Use tools
  const result = await zola.tools[0].execute({ path: "./my-site" });
  console.log(result);
}
----

=== Available Tools per Adapter

Each adapter typically provides:

* `<ssg>_init` - Initialize a new site
* `<ssg>_build` - Build the site
* `<ssg>_serve` - Start development server
* `<ssg>_check` - Validate site
* `<ssg>_version` - Get SSG version

== Architecture

=== Adapter Structure

Each adapter follows this pattern:

[source,javascript]
----
// Required exports
export const name = "SSG Name";
export const language = "Language";
export const description = "Description";

// Connection management
export async function connect() { ... }
export async function disconnect() { ... }
export function isConnected() { ... }

// MCP-compatible tools
export const tools = [
  {
    name: "ssg_operation",
    description: "Operation description",
    inputSchema: { type: "object", properties: { ... } },
    execute: async (params) => { ... }
  }
];
----

=== Security

All adapters implement input sanitization to prevent code injection:

* *Julia adapters*: Escape `\`, `"`, `$`, `` ` ``
* *Lisp adapters*: Escape `\`, `"`
* *Identifier validation*: Only alphanumeric and underscore
